# 万能视频下载器 - 本地运行指南

## 一、环境准备

### 1.1 必需软件

| 软件 | 版本要求 | 用途 | 下载地址 |
|------|----------|------|----------|
| Python | 3.8+ | 后端运行环境 | https://www.python.org/downloads/ |
| Node.js | 18+ | 前端运行环境 | https://nodejs.org/ |
| Git | 任意版本 | 代码管理 | https://git-scm.com/ |

### 1.2 检查环境是否就绪

打开终端（PowerShell 或 CMD），执行以下命令检查：

```powershell
# 检查 Python 版本
python --version
# 预期输出：Python 3.x.x

# 检查 pip 是否安装
pip --version

# 检查 Node.js 版本
node --version
# 预期输出：v18.x.x 或更高

# 检查 npm 版本
npm --version
```

如果以上命令都正常输出版本号，说明环境已就绪。

## 三、后端启动

### 3.1 创建 Python 虚拟环境（推荐）

```powershell
# 进入后端目录
cd D:\28707\free-video-downloader\backend

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows PowerShell:
.\venv\Scripts\Activate.ps1

# Windows CMD:
.\venv\Scripts\activate.bat

# 激活成功后，命令行前面会显示 (venv)
```

> **新手提示**：虚拟环境的作用是隔离项目依赖，避免不同项目之间的包版本冲突。

### 3.2 安装依赖

确保虚拟环境已激活（看到命令行前面有 `(venv)` 标记），然后执行：

```powershell
# 安装 requirements.txt 中的所有依赖
pip install -r requirements.txt
```

安装过程可能需要 1-3 分钟，取决于网络速度。看到 `Successfully installed ...` 表示安装成功。

### 3.3 配置环境变量

在项目根目录创建 `.env` 文件：

```powershell
# 确保在后端目录下
cd D:\28707\free-video-downloader\backend

# 创建 .env 文件
New-Item -Path .env -ItemType File
```

然后用文本编辑器（如 VS Code、记事本）打开 `.env` 文件，添加以下内容：

```env
# 必需：DeepSeek API Key（用于 AI 视频总结功能）
# 获取方式：访问 https://platform.deepseek.com/ 注册并创建 API Key
DEEPSEEK_API_KEY=your_deepseek_api_key_here

# 可选：Stripe 支付相关（当前免费版不需要）
# STRIPE_SECRET_KEY=sk_test_xxx
# STRIPE_WEBHOOK_SECRET=whsec_xxx
```

> **重要**：将 `your_deepseek_api_key_here` 替换为你真实的 DeepSeek API Key。
>
> 如果没有 DeepSeek API Key，AI 总结功能会报错，但视频下载功能仍可正常使用。

### 3.4 启动后端服务

```powershell
# 确保在后端目录且虚拟环境已激活
cd D:\28707\free-video-downloader\backend
.\venv\Scripts\Activate.ps1

# 启动服务
python main.py
```

成功启动后，你会看到类似以下的输出：

```
INFO:     Started server process [xxxxx]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

**后端服务已成功启动，监听端口 8000。**

> **注意**：保持这个终端窗口运行，不要关闭。如果需要停止服务，按 `Ctrl + C`。

## 六、常见问题排查

### 6.1 后端启动问题

#### 问题：端口被占用
```
OSError: [WinError 10048] 通常每个套接字地址(协议/网络地址/端口)只允许使用一次
```

**解决方案**：
```powershell
# 查找占用 8000 端口的进程
netstat -ano | findstr :8000

# 终止对应进程（将 xxxx 替换为实际的 PID）
taskkill /PID xxxx /F

# 或者修改后端端口，在 main.py 最后一行修改
uvicorn.run(app, host="0.0.0.0", port=8001)  # 改为 8001
```

#### 问题：缺少依赖
```
ModuleNotFoundError: No module named 'xxx'
```

**解决方案**：
```powershell
# 确保虚拟环境已激活
.\venv\Scripts\Activate.ps1

# 重新安装依赖
pip install -r requirements.txt
```

#### 问题：Python 版本过低
```
ERROR: Could not find a version that satisfies the requirement...
```

**解决方案**：升级 Python 到 3.8 或更高版本。

### 6.2 前端启动问题

#### 问题：端口被占用
```
Port 5173 is already in use
```

**解决方案**：
```powershell
# Vite 会自动尝试下一个可用端口，通常不需要处理
# 或者手动指定端口
npm run dev -- --port 5174
```

#### 问题：npm install 失败
```
npm ERR! code ENOENT
npm ERR! syscall open
```

**解决方案**：
```powershell
# 清除 npm 缓存
npm cache clean --force

# 删除 node_modules 重新安装
Remove-Item -Recurse -Force node_modules
npm install
```

### 6.3 功能异常问题

#### 问题：视频解析失败
- 检查 URL 是否正确
- 检查网络连接
- 查看后端终端是否有错误日志

#### 问题：AI 总结功能报错
- 检查 `.env` 文件中是否配置了 `DEEPSEEK_API_KEY`
- 检查 API Key 是否有效
- 检查视频是否有字幕（没有字幕无法总结）

#### 问题：抖音视频解析失败
- 抖音反爬策略经常更新
- 检查网络连接
- 稍后重试

## 七、开发模式说明

### 7.1 热重载

- **后端**：修改代码后需要手动重启（按 Ctrl+C 停止，再运行 `python main.py`）
- **前端**：Vite 支持热重载，修改代码后浏览器会自动刷新

### 7.2 前后端通信

- 前端运行在 http://localhost:5173
- 后端运行在 http://localhost:8000
- Vite 配置已将 `/api` 开头的请求代理到后端

### 7.3 调试技巧

#### 查看后端日志
后端终端会显示所有请求和错误信息，遇到问题先看日志。

#### 浏览器开发者工具
- 按 `F12` 打开开发者工具
- Network 标签查看 API 请求
- Console 标签查看前端错误

#### API 测试
可以直接在浏览器或使用工具（如 Postman）测试后端 API：

```
POST http://localhost:8000/api/parse
Content-Type: application/json

{
  "url": "https://www.bilibili.com/video/xxxx"
}
```
