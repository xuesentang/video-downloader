# 万能视频下载器 - 项目架构指南

## 一、项目概述

### 1.1 项目定位
这是一个基于 Python + Vue.js 的全栈视频下载应用，支持 1800+ 平台的视频解析与下载，并集成了 AI 视频总结功能。

### 1.2 核心能力
- **视频解析**：支持 YouTube、Bilibili、抖音、TikTok 等 1800+ 平台
- **视频下载**：提供直链下载和服务端代理下载两种模式
- **AI 总结**：基于 DeepSeek 大模型自动生成视频摘要、思维导图、智能问答
- **用户系统**：支持注册登录、VIP 权限管理（当前免费版已解锁全部功能）

## 四、后端架构详解

### 4.1 核心模块职责

#### 4.1.1 main.py - 应用入口
- **职责**：FastAPI 应用初始化、全局中间件配置、路由挂载
- **关键功能**：
  - CORS 跨域配置
  - 生命周期管理（启动初始化数据库、关闭清理临时文件）
  - 核心 API 端点：健康检查、视频解析、视频下载、直链获取、缩略图代理
  - 子路由挂载（认证、AI 总结）

#### 4.1.2 downloader.py - 通用下载器
- **职责**：基于 yt-dlp 的通用视频下载封装
- **核心类**：`VideoDownloader`
- **关键方法**：
  - `parse_video(url)`：解析视频信息，返回统一格式
  - `download_video(url, format_id)`：服务端下载视频
  - `get_direct_url(url, format_id)`：获取视频直链
  - `_extract_formats(info)`：提取并整理可用格式列表

#### 4.1.3 douyin.py - 抖音专用解析器
- **职责**：抖音视频的无 Cookie 解析方案
- **核心类**：`DouyinParser`
- **技术原理**：
  1. 短链接重定向解析
  2. 从 URL 提取 video_id
  3. 调用公开 API 获取元数据
  4. 获取无水印播放地址
- **反爬处理**：WAF 验证自动绕过、请求头伪装、重试机制

#### 4.1.4 summarizer.py - AI 总结模块
- **职责**：字幕提取 + DeepSeek AI 分析
- **核心类**：
  - `SubtitleExtractor`：从视频提取字幕（支持 YouTube、Bilibili、抖音）
  - `VideoSummarizer`：调用 DeepSeek API 生成总结
- **字幕获取策略**：人工字幕 > 自动字幕 > 标题描述（抖音）

#### 4.1.5 database.py - 数据库层
- **职责**：SQLite 数据库操作封装
- **数据表**：
  - `users`：用户信息、VIP 状态
  - `orders`：订单信息、支付状态
- **关键特性**：WAL 模式、外键约束、上下文管理器

### 4.2 API 路由设计

| 端点 | 方法 | 功能 | 模块 |
|------|------|------|------|
| `/api/health` | GET | 健康检查 | main.py |
| `/api/parse` | POST | 解析视频信息 | main.py |
| `/api/download` | POST | 服务端下载视频 | main.py |
| `/api/direct-url` | POST | 获取视频直链 | main.py |
| `/api/proxy/thumbnail` | GET | 代理获取缩略图 | main.py |
| `/api/auth/register` | POST | 用户注册 | api_auth.py |
| `/api/auth/login` | POST | 用户登录 | api_auth.py |
| `/api/auth/me` | GET | 获取当前用户 | api_auth.py |
| `/api/summarize` | POST | AI 视频总结（SSE 流式） | api_summarize.py |
| `/api/chat` | POST | AI 视频问答（SSE 流式） | api_summarize.py |

### 4.3 数据流转路径

```
用户输入 URL
    ↓
[main.py] 路由分发
    ↓
判断平台类型
    ├─ 抖音 → [douyin.py] 专用解析
    └─ 其他 → [downloader.py] yt-dlp 解析
    ↓
返回统一格式的视频信息
    ↓
用户选择下载/AI总结
    ├─ 下载 → 直链/服务端下载 → 文件传输
    └─ AI总结 → [summarizer.py] 提取字幕 → DeepSeek API → 流式返回
```

```javascript

// 视频状态
```

### 5.3 API 封装层

所有 API 调用封装在 `src/api/` 目录下：

- `video.js`：视频解析、下载相关

- `summarize.js`：AI 总结、问答（SSE 流式处理）
- `payment.js`：支付相关（当前禁用）

### 5.4 响应式设计

- **断点设计**：移动端优先，适配 sm(640px)、lg(1024px) 等断点
- **布局策略**：
  - 视频结果页：移动端垂直堆叠，桌面端左右双栏（左 2/5，右 3/5）

## 七、关键设计决策

### 7.1 为什么选择 yt-dlp？
- **优势**：支持 1800+ 平台、持续更新、社区活跃
- **定位**：作为通用下载引擎，覆盖绝大多数平台

### 7.2 抖音为什么要单独处理？
- **问题**：yt-dlp 需要 Cookie，且抖音反爬严格
- **方案**：基于公开 API 的无 Cookie 解析方案
- **实现**：`douyin.py` 独立模块，WAF 自动绕过

### 7.3 AI 总结的字幕获取策略
- **优先级**：人工字幕 > 自动字幕 > 标题描述（抖音）
- **B站**：通过 `dm/view` API 获取 CC 字幕
- **抖音**：使用标题+描述作为内容源

### 7.4 免费版的设计
- **策略**：所有用户默认 VIP，无功能限制
- **标记**：代码中使用 `[FREE_VERSION]` 注释标记免费版逻辑
- **扩展**：保留支付模块，方便后续商业化

## 八、开发规范

### 8.1 代码规范
- **命名**：函数/变量使用 snake_case，类使用 PascalCase
- **注释**：关键逻辑必须带注释，说明「是什么、为什么、怎么用」
- **错误处理**：所有可能出错的地方必须有 try-except 包裹

### 8.2 版本标记
- `[FREE_VERSION]`：免费版特定逻辑
- `[PAID_VERSION]`：付费功能（当前注释掉）

### 8.3 环境变量
```bash
# 必需
DEEPSEEK_API_KEY=your_api_key_here

# 可选
STRIPE_SECRET_KEY=sk_test_xxx          # 支付功能（当前不需要）
STRIPE_WEBHOOK_SECRET=whsec_xxx        # 支付回调（当前不需要）
```

## 十一、常见问题排查

### 11.1 视频解析失败
- 检查 URL 是否有效
- 查看是否为受支持的平台
- 检查网络连接

### 11.2 AI 总结失败
- 确认 `DEEPSEEK_API_KEY` 已配置
- 检查视频是否有字幕
- 查看 API 调用配额

### 11.3 抖音解析失败
- 抖音反爬策略经常更新
- 检查 `douyin.py` 中的请求头是否需要更新
