# 万能视频下载网站 - 方案设计文档

## 1. 技术架构

```
┌──────────────────────────────────────────────────────────┐
│                     用户浏览器                            │
│  ┌─────────────────────────────────────────────────────┐ │
│  │           Vue3 + Vite + Tailwind CSS               │ │
│  │  URL输入 → 解析展示 → 格式选择 → 下载              │ │
│  │                     ↘ AI总结 → 摘要/字幕/导图/问答  │ │
│  └──────────────────────┬──────────────────────────────┘ │
└─────────────────────────┼────────────────────────────────┘
                          │ HTTP API / SSE
┌─────────────────────────┼────────────────────────────────┐
│                  FastAPI 后端                              │
│  ┌──────────────────────┴──────────────────────────────┐ │
│  │  /api/parse      → 解析视频信息                      │ │
│  │  /api/download   → 服务端下载并提供文件              │ │
│  │  /api/direct-url → 获取视频直链                      │ │
│  │  /api/summarize  → AI视频总结（SSE流式）             │ │
│  │  /api/chat       → AI视频问答（SSE流式）             │ │
│  └──────────┬─────────────────────────┬──────────────┘ │
│             │                         │                  │
│  ┌──────────┴──────────┐  ┌───────────┴──────────────┐ │

│  │   download()         │  │   → 字幕提取(yt-dlp/B站)  │ │
│  │   get_url()          │  │   VideoSummarizer         │ │
│  │                      │  │   → DeepSeek API 总结/问答 │ │
│  └──────────────────────┘  └────────────────────────────┘ │
└──────────────────────────────────────────────────────────┘
```

## 2. 技术栈选型

| 层次 | 技术 | 选型理由 |
|------|------|----------|
| 前端框架 | Vue 3 + Vite | 轻量、上手快、生态成熟 |
| CSS 框架 | Tailwind CSS | 快速构建精美 UI，无需写大量自定义 CSS |
| Markdown 渲染 | marked + @tailwindcss/typography | marked 转 HTML + prose 类精美排版 |
| 思维导图 | markmap-lib + markmap-view | 开源成熟，支持 Markdown 输入，交互性好 |
| 后端框架 | FastAPI (0.135+) | 异步高性能、原生 SSE 支持、Python 生态 |
| 核心能力 | yt-dlp | 14w+ Star，支持 1800+ 网站，社区活跃 |
| AI 大模型 | DeepSeek (deepseek-chat) | 兼容 OpenAI 协议，通过 openai SDK 调用 |
| 数据库 | 无/SQLite | 轻量设计，无需持久化存储 |

### 3.2 下载视频（服务端代理模式）

```
POST /api/download
Content-Type: application/json

Request:
{
    "url": "https://www.youtube.com/watch?v=dQw4w9WgXcQ",
    "format_id": "137+140"
}

Response: 视频文件流 (Content-Type: video/mp4)
Headers:
  Content-Disposition: attachment; filename="视频标题.mp4"
```

### 3.3 获取直链

```
POST /api/direct-url
Content-Type: application/json

Request:
{
    "url": "https://www.youtube.com/watch?v=dQw4w9WgXcQ",
    "format_id": "137"
}

Response (200):
{
    "success": true,
    "data": {
        "direct_url": "https://rr3---sn-xxx.googlevideo.com/...",
        "ext": "mp4",

    }

```

### 3.4 AI 视频总结（SSE 流式）

```
POST /api/summarize
Content-Type: application/json

Request:
{
    "url": "https://www.bilibili.com/video/BV1mAAmzqEfP",
    "language": "zh"
}

Response: SSE 事件流 (Content-Type: text/event-stream)
事件类型：
  - subtitle  → 字幕数据（含 segments 时间戳列表和 full_text）
  - summary   → 总结摘要（流式 token，逐字输出）
  - mindmap   → 思维导图 Markdown（一次性返回）
  - done      → 完成信号
  - error     → 错误信息
```

### 3.5 AI 视频问答（SSE 流式）

```
POST /api/chat
Content-Type: application/json

Request:
{
    "url": "https://www.bilibili.com/video/BV1mAAmzqEfP",
    "question": "这个视频用了什么技术栈？",
    "subtitle_text": ""
}

Response: SSE 事件流 (Content-Type: text/event-stream)
事件类型：
  - answer    → AI 回答（流式 token）
  - done      → 完成信号
  - error     → 错误信息
```

## 4. 下载模式策略

系统自动选择最优下载模式：

1. **优先尝试直链模式**：调用 `/api/direct-url` 获取直链
   - 如果成功且直链可用 → 浏览器直接下载（不占服务器带宽）
2. **回退到代理模式**：如果直链不可用或有防盗链
   - 调用 `/api/download` → 服务端下载后流式传输给用户

## 7. AI 视频总结模块设计

### 7.1 字幕提取策略

```
用户输入 URL
    ├─ B站视频？ → B站专用 API（x/v2/dm/view）获取 CC 字幕 / AI 字幕
    │              ├─ 成功 → 返回字幕
    │              └─ 失败 → 回退到 yt-dlp
    └─ 其他平台 → yt-dlp 提取字幕
                   ├─ 人工字幕（优先：zh > en > ja > ko > 其他）
                   ├─ 自动字幕（同上优先级）
                   └─ 无字幕 → 返回提示
```

字幕格式：VTT → 解析为结构化分段 `[{start, end, text}, ...]`

### 7.2 AI 调用流程

1. 提取字幕文本（截断至 15000 字符）
2. 调用 DeepSeek API（stream=True）流式生成总结摘要
3. 调用 DeepSeek API 生成思维导图 Markdown
4. 通过 SSE 实时推送给前端

### 7.3 SSE 数据传输

- **summary / answer 事件**：token 通过 `json.dumps()` 编码后发送，避免换行符被 SSE 协议拆行导致前端丢失格式
- **前端解析**：严格按照 SSE 规范处理，`data:` 行用 `\n` 拼接，空行触发事件派发，`summary` / `answer` 数据先 `JSON.parse` 解码

### 7.4 前端渲染

- **总结摘要**：marked 解析 Markdown → `@tailwindcss/typography` prose 类排版 + 自定义样式覆盖（h1/h2/h3/blockquote/code/pre/table/a 等），流式打字机效果
- **字幕文本**：带时间戳的列表，支持展开/收起，支持下载字幕文件（SRT / VTT / TXT 三种格式）
- **思维导图**：markmap-lib 转换 Markdown → markmap-view 渲染 SVG，支持全屏展示（Fullscreen API），支持下载高清 PNG（4K+）和 SVG 矢量图
- **AI 问答**：对话式 UI，流式输出 + 光标闪烁动画，AI 回复使用 prose 排版

### 7.5 思维导图导出策略

- **PNG 导出**：通过 `<g>` 根节点的 `getBBox()` 获取完整内容边界（不受用户缩放/平移影响）→ 将 `foreignObject` 替换为 SVG `<text>` 元素（规避 Canvas tainted 安全限制）→ 设置完整 `viewBox` + 4x 以上缩放 → Canvas 绘制 → `toBlob()` 导出
- **SVG 导出**：克隆 SVG DOM → 通过 `getBBox()` 重设完整 `viewBox` → 内联 markmap CSS → 序列化为独立 SVG 文件
- **全屏展示**：使用浏览器原生 Fullscreen API，全屏后调用 `markmapInstance.fit()` 自适应
